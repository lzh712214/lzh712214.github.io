win10下vs2013+openGL+openmesh配置

配置

1.下载对应版本的openGL和openmesh

2.将下载好的openGL解压。

我下的压缩文件里会有两个.dll文件，两个.lib文件和一个.h文件。将两个.dll文件放到c:\windows\syswow64里。32位的电脑放到system32里。将两个.lib放入..\vs2013\vc\lib里，将.h放入vc\include\GL里，没有GL则新建。

3.将openmensh安装

之后如果要用到openmensh，就在工程的解决方案属性里把include和lib包含进去。

遇到的问题

​ 我是在打开别人的工程并编译时遇到问题的。首先，因为他的工程中用了qt，所以要先配置好qt。然后在打开工程是，要将解决方案-qt project settings 中的qt版本设置好。其次要配置包含目录和链接库。开始我是在vc++目录里配置的，发现没用。百度之后说，要在c/c++里配置，在附加包含目录里将openmesh的include和qt的include都包含进去，在链接器-》常规里把附加目录库，也是include包含进去，然后在链接器-》输入里把附加依赖项都包含进去。该工程的附加依赖项如下：

qtmaind.lib
OpenMeshCored.lib
OpenMeshToolsd.lib
Qt5Core.lib
Qt5Gui.lib
Qt5OpenGL.lib
Qt5Widgets.lib
Qt53DInput.lib
glut.lib
opengl32.lib
glu32.lib

按道理来说，如上配置调试是没有问题的。但是，报错了，一堆的xx文件未找到。然后我就在c/c++的附加包含目录里，将没要找到的都文件放了进去。如：原本是C:\qt\qt5\qt5.71\msvc2013_64\include 详细到C:\qt\qt5\qt5.71\msvc2013_64\include\Qwidgets.

我遇到的报错一般是无法解析外部符号。这种就是链接库有问题，即.lib的包含问题。

还有是xx文件未找到（no such file ….)。这种将文件正确的包含进去就行。

还有一种就是项目计算机x86不匹配目标计算机x64。这个问题是将open GL的.h即glut.h，从解决方案里放入包含目录里时出现的问题。很简单，不要包含，就不报错。但如果不包含，他就会报一堆xxx符号无法解析之类的错。这里有一个问题，就是我们在将qt project settings 设置以后，原来的附加依赖项变少了。然后就导致了后面的一堆问题。最简单的方法就是，将上面的.lib文件直接复制粘贴进附加依赖项里。前提是，c/c++里的附加包含目录写对。
